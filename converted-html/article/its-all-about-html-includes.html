It's all about HTML & Includes

Posted: Sat Mar 30 2013

Over the last few months I’ve read various conversations on Twitter and listened to podcast discussions regarding HTML and specifically template engines, the main gist of this conversation…

“I just want to use HTML but with includes”

…and…

“I don’t want to learn a new template language”

And of course we’ve had similar comments directed at us and our choice of template language for Mixture - Liquid.

In these cases it feels like there is an assumption that the Liquid template language is going to be complex or a learning curve that they just don’t have time for.

At first glance any new language can seem like hassle, and Liquid has a lot more features than just “includes”. To be honest it’s up to you how much of that power you want to use, if you just want HTML with the ability to use includes, then I think Liquid + Mixture are perfect. But, the best bit is - if you do need more, it has what you need.

We looked at a range of template languages before we chose Liquid, but the three main reasons that we did…

  1. It’s simple & familiar - it’s basically just HTML (+ some).
  2. You can use the basics and move on at your own pace.
  3. It’s extendible, so we can add features to aid development in Mixture.

Example

Here’s an some basic HTML…

<!doctype html>
<html>
  <body>
    <h1>Page Headline</h1>
  </body>
</html>

Here’s the same markup with an include…

<!doctype html>
<html>
  <body>
    {% include navigation %}
    <h1>Page Headline</h1>
  </body>
</html>

Which could load in something like this…

<nav>
  <a href=/>Home</a>
  <a href=/about>About</a>
</nav>

Of course I would definitely recommend you look deeper and take advantage of features like “layouts” as it will totally improve your structure and workflow.
