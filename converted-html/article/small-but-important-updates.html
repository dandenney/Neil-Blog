Small but important updates

Posted: Thu Apr 04 2013

If you’re using Mixture you may have noticed we are pushing updates continuously, improvements, fixes and new features.

We’ve just made another small but powerful update.

Mixture currently has two project types (or modes) “Mixture” and “Simple”.

A Mixture project benefits from all of the features and functionality that the tool and online services offer, builtin server, friendly urls, liquid templates, HTML, preprocessing, minifcation, live refresh, multi-device testing, quick publish and so on.

The idea behind simple mode was to allow non-mixture projects to benefit from some of the same features detailed above, so we built a feature into simple mode that allows Mixture to partner with any server (mamp, wamp, etc) and automatically enable live refresh, style injection and multi-device testing.

We think this is pretty powerful and very flexible.

But we’ve had feedback from users, who didn’t necessarily want to jump into a Mixture project, but also didn’t have a server running for their project. Usually thats because the project they are building is simply static HTML files.

Although it’s pretty straight forward to convert a project into a Mixture project, we thought we could make it easier to run these types of projects in Mixture.

So as of today and the latest update…

If you use Mixture in simple mode and don’t set a partner server, Mixture will boot a lightweight version of its own server. This server will parse HTML files and assets, it will even add live refresh, style injection and multi-device ability automatically!

As an example, if in the root of your project directory you had a file called “about.html” you could simply “view locally” on Mixture and then go to the url “/about” to view that page. Any changes to styles or the page will be automatically visible.

We really hope you like this small addition.
